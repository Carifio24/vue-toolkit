Creating a Story

Set up via template

In this guide we'll walk you through the process of creating a Vue-based data story. Whether you have some experience with Vue or are completely new to it, this guide will hopefully make the process of setting up your own data story a little bit easier.

To help you get started, the Cosmic Data Stories team has created a Vue DS template that gives you the skeleton of a fully working data story. The template creates a story built around WorldWide Telescope, but in the future we'll provide templates for other sorts of data stories as well.

To get started with the template, you'll want to use Git to clone the template. One way to do this is to open a terminal prompt and run

git clone https://github.com/cosmicds/vue-ds-template.git <your-story-name>

where <your-story-name> is the name of your story. Next, we have a setup script that you can run to help rename relevant pieces of the story:

  cd <your-story-name>
  scripts/setup.sh <your-story-name>

With the basic renaming done, there are a few commands that you can use while developing the story:

  • yarn serve
    runs the Vue development server. This will run a small development server that you can use to look at what the story currently looks like. This server uses "hot reloading", which means that updates you make to component files will automatically be reflected on the page.
  • yarn build
    - when you're ready to deploy a version of your story, run this command. It will build a static version of your story into the "dist" directory. You can then upload this directory onto a static file server (GitHub Pages works great) and you're good to go!
